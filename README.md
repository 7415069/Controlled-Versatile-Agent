## 🚩 宣言：智能，不应是预设的剧本

在 Agent 概念喧嚣的今天，我们遗憾地看到，大量所谓的"智能体"正沦为**披着 AI 外衣的传统自动化脚本**。

开发者们小心翼翼地预设了一套套 SOP（标准作业程序），在固定的节点调用大模型进行"填空"。这种做法虽然在特定场景下有效，但却在无形中阉割了大模型最珍贵的特质：**自主规划与动态适配的能力**。

**CVA 认为：**

* **智能不应被"喂养"**：如果一个 Agent 的能力边界取决于程序员写了多少行 `if-else`，那它只是一个高级的宏程序。
* **底座应是"无相"的**：底座不应包含任何业务逻辑，它只需提供受控的权限、持久的记忆和原子化的工具。
* **大脑是唯一的驱动**：只要 Token 足够充裕，模型足够聪明，人类只需指明终点，路径应由大脑在数字世界中自行开辟。

## 🧐 那些被"剧本"束缚的时刻

当现实的需求稍稍偏离了开发者预设的轨道，那些"伪智能体"往往会显露出其逻辑的底色：

* **当"地图"消失时**：面对一个从未见过的私有协议或文件格式，预设了 `ls` 和 `cat` 的 Agent 会陷入无尽的报错循环。因为它在等待人类为它更新"插件"，而不是思考如何自制一把钥匙。
* **当"路径"受阻时**：在长程任务中，环境是瞬息万变的。死守 Step 1 到 Step N 的 Agent，在第一步出错时就会满盘皆输。真正的智能，应如水之无常形，在每一轮迭代中自省、反思并重绘蓝图。
* **当"自由"与"安全"冲突时**：为了安全，我们往往选择禁锢。但禁锢了操作，也禁锢了可能。我们需要的不是一个被锁在笼子里的打字员，而是一个在执行高危操作前，懂得停下来向你阐述理由并申请"提权"的数字合伙人。

## 🧠 CVA 设计哲学：三层分离，大脑驱动一切

市面上大多数 Agent 框架是两层结构：**模型 + 工具**。CVA 认为这不够，它把架构拆成三层，每一层的职责边界严格分离：

```
┌─────────────────────────────────────┐
│         大脑（模型）                  │  ← 唯一的决策者，路径由它实时计算
├─────────────────────────────────────┤
│      状态基础设施（记忆 / 反思）        │  ← 补偿模型无状态的客观局限
├─────────────────────────────────────┤
│      原子底座（工具 / 权限 / 审计）     │  ← 物理法则，与任何业务无关
└─────────────────────────────────────┘
```

**为什么需要中间这一层？**

今天的大模型每次对话都是无状态的——它不记得上一轮做了什么，不知道任务做到哪了，不知道自己是谁。状态基础设施的存在，是为了补偿这个技术局限，把"你是谁、你在做什么、做到哪了"持续注入给模型。这不是业务逻辑，是让大脑能正常工作的基础条件。**当模型有了真正的持久记忆，这一层可以整体删掉。**

底座则是纯粹的物理法则。它不知道也不关心任务是什么，只知道：这个路径不能访问、这个命令需要申请、这个操作要记录在案。**换一个角色 YAML，底座行为不变，变的只是权限边界。**

### 通用底座 (Universal Shell)

底座退化为纯粹的"物理法则"，与任何具体业务完全解耦：

* **原子工具集**：提供文件、网络、进程、GUI 控制等最基础的"手"和"眼"。
* **安全边界 (Security Boundary)**：基于模式匹配与动态审计的权限护城河，拦截危险操作是无条件的——不管大脑怎么下命令。
* **全程审计 (Audit Log)**：记录大脑的每一次思考与动作，结构化、可追溯。

### 100% 外置的"智能驱动"

底座不写任何业务逻辑，路径完全由模型实时决策：

* **主动实施**：模型不再是被动问答，而是任务的主动执行者。
* **自我进化 (Self-Evolution)**：通过 `synthesize_tool`，模型在发现现有工具不足时，可自主编写 Python 工具并即时热加载，无需人类干预。
* **语义脱水 (Memory Dehydration)**：当上下文过长时，自动将旧内容压缩成语义骨架（类名、函数签名），让大脑在有限的 Token 窗口内依然能维持对复杂工程的全局记忆。

## 🥊 思想层级的博弈

OpenClaw（前身 Clawdbot）是 2026 年初席卷全球的现象级开源 Agent，在中国更是引发了"养龙虾"热潮——腾讯总部门口排起装机长队，77 岁老人也在排。但与此同时，研究人员发现其存在大量安全漏洞，付钱装完又付钱卸载的闹剧，恰好印证了"权限裸奔"的代价。

| 维度 | **MCP / Claude Code** | **OpenClaw** | **CVA** |
|:---|:---|:---|:---|
| **核心本质** | 标准化的**工具协议层**，解决模型与外部服务的连接问题 | **消息驱动的通用 Agent**，模型通过 IM 接收指令并自主执行任务 | **自主进化的数字人**，底座无逻辑，路径全靠大脑实时计算 |
| **工具获取** | **静态接入**。必须由人类先写好 MCP Server，模型才能获得新能力 | **Skills 系统**。技能以目录形式存储，社区贡献为主，缺乏安全审查 | **动态合成**。大脑意识到工具缺失，现场写代码造工具并原地进化 |
| **权限模型** | 工具级别的开关，缺乏细粒度的动态提权与实时审计 | **高危裸奔**。需要邮件、日历、文件系统的广泛权限，prompt injection 可直接劫持 Agent 行为 | **动态提权 (Escalation)**。高危操作自动挂起，向人类发起带理由的申请，全程审计 |
| **安全边界** | 依赖 MCP Server 自身的权限控制 | 第三方 Skill 无审查机制，已被证实存在数据外泄和恶意注入风险 | 权限最小化原则，危险操作必须人类审批，审计日志不可篡改 |
| **长程记忆** | 依赖上下文窗口，任务稍长容易丢失架构级视野 | 本地持久化历史，但无语义压缩，长任务下 Token 消耗快速膨胀 | **语义脱水**。自动压缩历史，保留架构骨架，支持超长任务 |
| **角色切换** | 换工具集需要重新接入 Server | 换任务场景需要重新配置 Skills 和权限 | 换一个 YAML 文件，模型立即切换身份、权限边界和工作范围 |

> **OpenClaw 证明了市场对"AI 真正做事"的渴望；CVA 要解决的，是"做事"与"失控"之间的那道防线。**

## 🛠️ 核心功能亮点

* **🛡️ 动态权限提权 (Escalation Manager)**：当模型触碰敏感路径或高危命令时，自动挂起任务，向人类发起带理由的申请。风险分级（LOW / MEDIUM / HIGH）由角色 YAML 定义，底座本身不做任何业务假设。

* **📉 智能记忆脱水 (Memory Store)**：当对话过长时，系统自动将旧代码内容"脱水"成语义骨架（类名、函数签名），既保留了架构记忆，又大幅降低 Token 消耗。截图等大型媒体内容单独处理，不参与 token 预算计算，不会因为一张截图就把整个操作历史裁掉。

* **🧬 工具合成引擎 (Synthesize Tool)**：模型发现工具缺失时，可现场生成 Python 工具类并热加载，无需重启，无需人类干预。工具合成完成后自动注册到工具表，下一轮即可使用。

* **🕵️ 全程审计日志 (Audit Log)**：每一轮思考、每一个工具调用、每一次权限变更，全部结构化记录为 JSONL 格式，支持日志轮转和过期清理，确保 Agent 行为可追溯、可审计。

* **🎭 角色驱动，零业务耦合**：底座不包含任何角色相关的判断。通过 YAML 定义角色的身份、初始权限和提权策略，换一个文件就换一个角色——从开发者到运维到 GUI 操作员，底座代码一行不改。

## 🚀 快速开始

### 1. 环境准备

```bash
git clone https://github.com/7415069/Controlled-Versatile-Agent
pip-compile --no-emit-index-url --no-emit-trusted-host requirements.in
pip install -r requirements.txt
```

### 2. 定义角色

在 `roles/` 目录下通过 YAML 定义你的角色。只需指明身份、初始权限和提权策略，**严禁写入任何业务逻辑**。

```yaml
role_name: developer-v1
identity_prompt: "你是一名资深后端开发工程师..."
init_permissions:
  read: ["./**"]
  write: ["./agent_workspace/**"]
  shell: ["python3", "pip", "git"]
escalation_policy:
  auto_deny_patterns: ["/etc/**", "~/.ssh/**"]
  low_risk_prefixes: ["./src", "./tests", "./docs"]
  notify_channel: console
  timeout_seconds: 300
capabilities:
  - read_file
  - write_file
  - run_shell
  - synthesize_tool
```

### 3. 唤醒大脑

```bash
# 启动 GUI 控制台
python cv_agent.py

# 或者纯 CLI 模式
python -m brtech_cva.cv_agent --manifest roles/developer-v1.yaml
```
